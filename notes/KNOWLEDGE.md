# kakao_manager 지식베이스 (단톡방별 특징·주의·참고 기억)

> 여러 Mac이 이 스킬을 공유한다. 이 파일은 **git push/pull로 공유되는 운영 지식**이다.
> ⚠️ **여기에 절대 적지 말 것**: 고객/수취인 PII(이름·전화·주소), 자격증명, 비밀번호, 대량발송 타겟 실데이터.
>   → 그건 레포 밖(`~/.config/kakao_manager/config.json`, `~/.config/kakao_manager/targets/`)에 둔다.
> 여기엔 **"방의 용도·열 때의 함정·발송/운영 규칙·재사용 노하우"** 만 적는다.
> 새 방/규칙 알게 되면 이 파일에 한 줄 추가하고 commit+push → 다른 맥은 pull로 받는다.

## 송장 운영 규칙 (쿠팡 가구매)
- **당일출고 컷오프 = 오후 2시.** 당일출고 필요 건 = ① 어제 주문분(쿠팡 상태 상품준비중/INSTRUCT) + ② 당일 **오후 2시 이전** 결제완료(ACCEPT). **오후 2시 이후 결제완료는 내일 발송** → 그날 송장 안 와도 됨.
- **송장 미전달 시 리브리 방에 독촉**: ⚠️ **건수·수취인 명단은 빼고** "어제 주문 건 / 당일출고 필요한 건 **금일자 송장 전달** 부탁"만 보낸다(현빈 지시).
- **택배사 추측 금지**: 시트명 신뢰 말 것(실증: 시트명 "한진택배 주문요청서"였지만 실제 택배사는 **CJ대한통운**). 미지정이면 사용자에게 확인. 무인이면 이미 배송지시(DEPARTURE)된 동일 시리즈의 deliverCode로 교차검증.
- 송장 파일은 「주문요청서」 시트의 **송장 열**에 채워져 온다. **첫 몇 행이 비어 있어도 아래쪽은 채워졌을 수 있으니 전 행 스캔**할 것(낚이지 말 것).

## 계정 전환 주의
- 부계정(예: **고야매니저**) 로그인은 **2차 보안인증(모바일 카톡 인증번호) + "다른 기기에서 로그인 중 → 강제 로그아웃"** 을 요구한다 → **무인 자동화 불가, 사용자 수동 인증 필요.**
- 카톡 Mac 앱은 **한 번에 한 계정**. 부계정 확인하려면 현재 계정 로그아웃(메뉴 `카카오톡 → 로그아웃`) 후 부계정 로그인. 되돌릴 수 있으나 재로그인도 2FA 뜰 수 있음.

## 일반 발송/운영 주의 (재확인)
- ★★**그날 첫 메시지는 인사 선행 (현빈 2026-07-15, 반드시)**: 특정 방에 **그날의 첫 발송**이면 반드시 `안녕하세요 담당자님,` 정도의 간단한 인사로 시작한 뒤 본론(전달/요청)으로 넘어간다. 인사 없이 본론부터 나가지 말 것(두장군 리브리 전달 때 인사 생략으로 피드백 받음). 같은 날 이어지는 후속 메시지는 인사 생략 가능.
- **서명 금지**: 항상 `--no-signature` (vendored kakao_send.py SIGNATURE="" 패치됨).
- **작업 후 대화창 닫기**: `--close` 또는 Cmd+W. 메인 '카카오톡' 창은 유지. 검색어도 클리어.
- **`hide all apps` 절대 금지**: Claude Code 터미널까지 hide돼 작업 불가.
- **송신 전 (나) 검증**: AXImage description='badge me' 로 본인 채팅 확인(동명이인 오발송 방지).

## 리브리 송장 파일 처리 (실전 2026-06-23)
- **송장 파일명 패턴**: `N.NN 리브리 슬개골 명단.xlsx` (이민준이 "전달드려요!"와 함께 첨부). 다운로드: `download_file.py "리브리" "6.23 리브리 슬개골 명단"` → ⚠️ 저장패널(AXDialog) 뜨면 `osascript -e 'tell application "System Events" to keystroke return'`로 마무리(기본위치 ~/Downloads).
- **시트 구조**: 「슬개골 명단」 시트, 5열 = 도서 / 받으시는분 / 전화 / 받는분주소1 / **송장**. (별도 「진행 안내사항」 시트 있음)
- ★**송장 열 전 행 스캔 필수**: 위쪽 행들 송장 비어 있어도 **아래쪽은 채워져 있음**(실증: 85명 중 위40 빈칸, 아래45 송장기재). 앞부분만 보고 "송장 없음/명단 불일치" 단정 금지(실수한 적 있음).
- **쿠팡 송장입력 매칭**: Wing 상품준비중(INSTRUCT) 수취인명을 **명단 전체**와 대조 + **주소 교차검증**(동명이인 방지). 송장 채워진 명단 행만 입력. 나머지(ACCEPT 등 다른 상태)는 상품준비중 될 때 추가. 입력 방법은 [[coupang-manager/modules/login.md + invoice.md]].
- **강제 로그아웃 모달**: 리브리 읽기/전송 전 atomacos로 '확인' 버튼 Press(고야매니저 폰 경합). read 뷰는 스크롤 위치가 불안정 → 파일첨부는 화면 스샷 또는 atomacos로 `.xlsx` static text 탐색으로 확인(kakao_read 텍스트엔 파일 안 잡힘).
- ★**파일 다운로드 함정 (2026-06-24, 1시간 삽질)**: 파일 버블이 **화면 하단/가장자리에 있으면 우클릭 컨텍스트 메뉴가 안 뜬다**(아래로 펼칠 공간 부족). `download_file.py`가 "저장하기 메뉴 못 찾음"으로 실패. → **먼저 파일을 창 중앙으로 스크롤**: 마우스를 대화영역에 두고 `Quartz.CGEventCreateScrollWheelEvent`(line 단위)로 파일이 창 중앙(y≈창중앙)에 올 때까지 스크롤(부호는 실측, 보통 +3 몇 번). 그 다음 파일 버블 좌표 우클릭(CGEvent) → 컨텍스트 메뉴의 "저장하기" 클릭. **컨텍스트 메뉴는 atomacos AXMenuItem로 안 잡히지만 trusted 우클릭 후엔 화면에 떠서 atomacos로 메뉴항목('저장하기','대화 캡처' 등) 탐색·클릭 가능.** 저장패널 뜨면 Return.
- ★**카톡 메인창 안 보일 때 (2026-06-24)**: 카톡은 트레이 상주라 메인창을 닫으면 `atomacos app.windows()`가 0개로 나온다. 게다가 메인창이 **다른 데스크탑(Space)에 있으면** `open -a`/`reopen`이 AppleEvent 타임아웃(`-1712`)으로 먹통. → `selftest.py`가 "메인창 안 보임 WARN"이면 **`killall KakaoTalk` 강제종료 후 `open -a KakaoTalk`로 재시작 + config 자동로그인(§2.1)**. 재시작 시 로그인창 뜨면 자동로그인(신규회선 계정은 2FA 없이 통과).

## 전면화·정확오픈·발송검증 노하우 (실전 2026-07-15 두장군 리브리 전달, 실제 유효했던 것만)
- ★**전면화 = Dock 아이콘 클릭(한글명 "카카오톡")이 정답**: 카톡이 트레이 상주/다른 Space에 있으면 `tell application "KakaoTalk" to activate`·`open -a KakaoTalk`가 **안 먹는다**(frontmost가 안 바뀜, `open -a`는 -600/-1712). ★확실히 되는 방법 = **Dock 클릭**: `osascript -e 'tell application "System Events" to tell process "Dock" to click UI element "카카오톡" of list 1'`. ⚠️UI element 이름은 반드시 **한글 "카카오톡"** — 영문 "KakaoTalk"은 `-1728`로 실패. 앱의 Space를 전면화+frontmost로 끌어옴(§57 killall 재시작보다 가벼운 1순위 방법).
- ★**open_exact 전제 = 카톡이 frontmost일 것**: `open_exact.py`는 Cmd+F를 쓰는데, 카톡이 전면이 아니면 **Cmd+F가 앞의 다른 앱(Chrome 등)으로 새서** 그 앱 찾기창에 "리브리 0/0"만 뜨고 open_exact는 `ok:false`(방 목록은 atomacos로 읽혀 candidates는 나옴). → open_exact 직전에 **위 Dock 클릭으로 전면 확정** 후 실행. 성공 시 `{"ok":true,"opened":"리브리"}`(stray stop_리브리 창은 헬퍼가 자동으로 닫음).
- ★**정방/오방 최종검증 = 멤버목록 팝업**: open_exact로 방 연 뒤, 창 **제목 아래 멤버수(👤N)를 클릭**하면 멤버목록 팝업이 뜬다 → `고야 한수지`·`이민준` 있으면 정방, `stepred` 있으면 오방(stop_리브리). (2026-07-15 실측 정방 리브리 멤버 = **우리계정(CLD)+`• 이민준`+`고야 한수지` 3명**. ⚠️햄버거 `≡`는 '채팅방 나가기' 포함 메뉴라 멤버확인엔 부적합+오클릭 위험 → 멤버수 클릭을 쓸 것.)
- ★**외부 거래처 발송 전 evaluator 게이트**: 스샷 2장(방 내용 + 멤버목록)을 evaluator 서브에이전트에 넘겨 [제목=정확히 "리브리"·멤버 고야한수지/이민준 존재·stepred 부재·대화맥락 일치] 판정 → `safe_to_send=true`만 발송. 조금이라도 불확실하면 발송 보류+보고(오방 발송 방지).
- ★**발송확정 검증 + Enter 함정**: 발송 성공 신호 = **입력창 비워짐 + "전송" 버튼 회색**. ⚠️**Enter가 발송이 아닐 수 있다** — 멘트+링크를 개행 포함으로 붙여넣으면(Cmd+V) Enter는 **줄바꿈으로 먹혀** 미전송된다 → **노란 "전송" 버튼을 좌표 클릭**해야 실제 발송. 발송 후 방에 **노란 말풍선** + **채팅목록 최상단 갱신**으로 재확인.
- ★**Drive 링크 검증**: 구글 드라이브 폴더 링크를 보내면 카톡이 **미리보기 카드("<폴더명> - Google Drive")** 를 자동 생성한다 — 카드가 뜨면 링크 유효·공유 정상. 단 상대가 열려면 **사전에 Drive 공유를 "링크 있는 사용자 열람가능(anyone-with-link reader)"** 로 설정해야 함(Drive API `permissions.create {type:anyone,role:reader}`).
- ★**발송은 좌표=화면포인트(displayed×1.28)** : screencapture(-x) 이미지는 retina 2배. Quartz 클릭 좌표(포인트) = 스샷 표시좌표 × 1.28.
- ★**채팅방이 더블클릭으로 안 열릴 때**(atomacos Quartz 더블클릭 무효, 통합/목록 모드): 방 행을 **싱글클릭(선택)한 뒤 Enter(`key code 36`)** 를 보내면 별도 대화창으로 열린다 → 창 목록에 방이름이 등장(`리브리, 카카오톡`). 이게 가장 확실.
- ★**`download_file.py`가 "채팅방 'X' 창이 없음 — 먼저 kakao_read.py로 방을 열 것" 에러** = 방이 **별도 창으로 안 열린 상태**. 위 Enter 방법으로 방을 별도창으로 띄운 뒤 실행해야 동작.
