# kakao_manager × 챗펄스 (카톡 적응형 상담 폴링)

> 챗펄스 패턴(`~/.claude/skills/챗펄스/SKILL.md`)의 **카톡 채널 구현**. 2026-07-15 신설(현빈 지시).
> 참조원본 = 여포 `chatpulse_growtalk.py`·`chatpulse_poller.py`(그로우톡). 여포 레포 커밋 올라오면 pull 후 대조·이식 정합화.

## 구성

| 파일 | 역할 |
|---|---|
| `scripts/chatpulse_kakao.py` | 채널 헬퍼: `open` / `lastmsg` / `read` / `send` (여포 chatpulse_growtalk 대응) |
| `scripts/chatpulse_poller_kakao.py` | 폴러: 맥박 스케줄 감지 → exit 0/2/3 (여포 chatpulse_poller 대응) |

## 맥박 스케줄 (챗펄스 공통)

- 마지막 대화(양쪽 아무거나)로부터 경과: **<5분=30초 / <30분=10분 / 그외=1시간**.
- 상대답장·내발신 = `last_activity` 리셋(→30초). 내발신 감지 시 폴러는 baseline만 갱신하고 **깨우지 않음**.
- 야간(22:00~09:00 KST)은 **1시간 모드일 때만 중지** → 폴러 `exit 3 (status=night_pause, resume_at=09:00)`. 에이전트가 09:00로 재스케줄(ScheduleWakeup/cron). 뜨거우면(30초/10분) 밤에도 계속.

## 폴러 계약 (exit 코드)

- **exit 0** = 상대 새 메시지 감지. stdout `{status:"new_message", room, last:{sender,text,time,is_mine}, last_fp, last_activity}`. → 에이전트가 맥락 읽고 **답장 문구 매번 생성**·발송 후, 새 `--last-fp`/`--last-activity`로 폴러 재기동.
- **exit 2** = 오류(카톡 사망/읽기 불가). → 에이전트가 복구(Dock 전면화·재오픈) 후 재기동.
- **exit 3** = 하트비트(max_run 8h 도달) 또는 야간중지. → 재시작/재스케줄 판단.

## 운용 흐름 (리브리 예)

```bash
# 1) 정확오픈(오방 배제) — 최초 1회
python3 scripts/chatpulse_kakao.py open "리브리"
# 2) baseline 확립
python3 scripts/chatpulse_kakao.py lastmsg "리브리"     # → fp 확보
# 3) 폴러 백그라운드 기동(run_in_background)
python3 scripts/chatpulse_poller_kakao.py --room "리브리" --last-fp <fp> --last-activity <epoch>
# 4) 폴러 exit 0(상대 새 메시지) → 에이전트 기상 → read/스샷으로 내용 확인 → 답장 생성 → send → 폴러 재기동
python3 scripts/chatpulse_kakao.py send "리브리" "<맥락 맞춘 답장>"
```

- ⚠️ **내용 읽기**: 카드/파일/미리보기 메시지는 `read`가 `text:null`로 잡을 수 있다(kakao_read 한계) → **스크린샷(screencapture)으로 내용 확인**(KNOWLEDGE 참조). 폴러의 감지(fp 변화)는 정상.
- ⚠️ **정방/오방**: `리브리`는 `open_exact`로만 열기(chatpulse_kakao `open`이 래핑). 발송 전 멤버(고야 한수지·이민준) 확인은 KNOWLEDGE의 evaluator 게이트 준용.

## 대화 규칙 (챗펄스 준수 — 카톡 발화 시 반드시)

- **이모지 금지**. **AI티 금지** — 발송 전 스레드 위쪽 말투 미러링(한 메시지 한 화제, 짧게 나눠 연타, 격식 스택 금지, 목록·번호 금지, 0초 즉답 금지=수십초 텀).
- **그날 첫 메시지 = 인사 선행**(`안녕하세요 담당자님,`) — KNOWLEDGE 규칙과 동일.
- **리액션/홀딩**: 상대가 답해주면 대화 마디 끝에 짧게 닫기("네 확인했습니다!"). 즉답 불가면 "확인해보고 말씀드릴게요" 먼저 치고 잠수 금지, 지연 시 중간 업데이트.
- ★**게이트(멈추고 현빈 확인)**: 결제/비용 확정 동의/신규 신청서 제출/계약성 발언 → **telegram-send + 지메일(coq3820@gmail.com) 양쪽 발송**으로 현빈 확인받고 진행. (챗펄스 게이트 알림에 한해 텔레그램 직접발송 허용 — 현빈 명시 예외)
- **대화 마무리 보고**: 대화 마디 종결 시 현빈 텔레그램 요약보고(무슨 얘기·어떻게 마무리·다음 예상). 폴러는 종료 말고 유지.
- **하루 1회 현황 브리핑**: 폴러 살아있는 동안 매일 1회 텔레그램(특히 상대 답변이 며칠 걸리는 대기상태 알림). 폴러 기상 시 날짜 바뀌었고 08시 이후면 발송.

## 첫 적용 대상 = 리브리(고야 한수지) 방

- **맥락**: 두장군 가구매 리뷰 30건 폴더 링크 전달(2026-07-15) 완료 → 리브리 회신 대응(수량확인·이상문의·견적요청 등).
- **감시대상**: 리브리 방 채팅. (배대지식 상태변화 감시는 이 방엔 해당 없음 — 채팅만.)
- **종료 정책**: 리브리 건 마무리(회신 해소 + 마무리 보고) 시 폴러 종료. 48h 무변화면 일 1회 체크로 강등.

## 종료 정책 (무한가동 금지, 챗펄스 공통)

1. 안건 완결 시 폴러 최종 종료.
2. 48h 무변화 → 상시폴러 종료, 일 1회 체크로 강등. 변화 재감지 시 재가동.
3. 8h 하트비트(exit 3)로 재시작 시점에 1·2 판정.
